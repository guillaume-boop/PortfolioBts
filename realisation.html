APPLICATION WEB







Ce que j'ai appris:
- Création d'une solution web pour répondre à une problématique
- Réfléxion sur comment automatiser par le code une action manuelle
- Manipulation des tables de prestashop


Technologies Utilisées:
- HTML, CSS et JS basique pour la page
- PHP pour l'envoi de requête sur le serveur distant Filezilla
- SQL pour l'interaction avec la base de donnée sur Phpmyadmin
- Prestashop



Développement d'une application web d'insertion de donnée dans des tables.
Dans le cadre de la création d'un site web de e-commerce pour une entreprise via un prestashop.
Et face à des problématiques multiples tel que des prix différends pour chaques clients ou le non visibilité des produits par les clients qui ne possèdent pas de prix spécifiques. Il fallais donc trouver des solutions via la manipulation du prestashop.
Cependant ces solutions prenaient un temps conséquent à faire manuellement depuis le prestashop.
Il fallait donc une fonctionnalité permettant d'effectuer toutes ces actions à l'aide du code.
On a donc créer des pages permettant d'effectuer ces actions sur les tables directement plutôt que sur le prestashop.

Projet de Stage de deuxième année de BTS SIO.

Développement d'un site web en local pour une entreprise fictive permettant la location de vélo et autres appareils.
- Création d'un formulaire pour les utilisateurs afin de se créer un compte.
- Création d'un login pour se connecter et d'un bouton pour se deconnecter.
- Création d'un bouton qui renvoie vers les pages qui permettent d'effectuer une reservation.
- Création d'une page qui permet au client de consulter, modifier ou supprimer sa commande.

Projet Scolaire sur le premier Semestre de deuxième année de BTS SIO
Première page de réservation

Page qui demande à l'utilisateur de renseigner l'agence où il souhaite réserver, l'heure et la date de début et de la fin de la reservation et le nombre de participant.
L'ensemble des champs est obligatoire et l'utilisateur doit être connecté à son compte pour acceder à cette page.
Le formulaire est soumis après que l'utilisateur prenne ou soumette une photo.

Deuxième page de réservation

Page qui demande à l'utilisateur de saisir le nom et prénom de chaque participant ainsi que le type de véhicule pour chacun en fonction du stock disponible en agence.
L'ensemble des champs est obligatoire (sauf celui des demandes spécifiques) et l'utilisateur doit être connecté à son compte pour acceder à cette page en plus d'avoir renseigné la page précédente.
Un bouton permet de revenir en arrière si l'utilisateur veut modifier les informations sur la page précédent.

Page récapitulative des commandes

Cette page est un récapitulatif des informations précédemment renseignées par l'utilisateur
L'utilisateur doit être connecté à son compte pour accèder à cette page et avoir renseigné la page précédente.
Un bouton permet de revenir à l'étape précédente si l'utilisateur veut modifier les informations sur la page précédente.
Le bouton "PAYER" permet de valider la commande en l'insérant dans le base de donnée.

Page de connexion

Page qui demande l'insertion de l'adresse mail et le mot de passe.
Si les deux sont correctes alors l'utilisateur est connecté sinon un message d'erreur s'affiche.

Formulaire de création de compte

Formulaire qui demande la saisie d'un nom, d'un prénom, d'une adresse mail et d'un mot de passe afin de créer un compte client.
Si l'adresse mail est déja reliée à un compte client alors l'utilisateur ne peut pas créer de compte.

Page de consultation des commandes

Page permettant à un utilisateurs connecté de rechercher une commande qu'il a validé grâce à l'id cette dernière.
Si l'id de la commande est bien rattaché à l'utilisateur connecté, alors un récapitulatif de la commande s'affiche et l'utilisateur peut la modifier ou la supprimer.
Si aucune correspndance n'est trouvée la recherche renvoie un message d'erreur.
